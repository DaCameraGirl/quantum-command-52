## Início rápido

Na raiz do repositório no PowerShell:

```powershell
python grants.py rank
python housing_violations.py summarize
python shell_catalog.py estimate
python quantum_portfolio.py --capital 1000
python qml_signal_engine.py --capital 1000
python strict_macro_quantum_v10.py --preflight
```

As saídas vão para a pasta `output`.

## IBM Quantum

O script rigoroso V10 exige dependências enterprise e acesso ao IBM Runtime.

```powershell
py -3.11 strict_macro_quantum_v10.py --preflight
```

Mantenha segredos em `.env`. Não faça commit de chaves privadas IBM, Alpaca, banco de dados ou outras.

### Pesquisa em papel Alpaca

Trading ao vivo é bloqueado intencionalmente. Apenas pré-visualização/envio em papel:

```powershell
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders --submit-paper-orders
```

## Configuração do painel web

```powershell
cd web-dashboard
npm install
py -3.11 -m pip install -r requirements.txt
py -3.11 server.py
```

Segundo terminal:

```powershell
cd web-dashboard
npm run dev
```

Abra http://127.0.0.1:5173

## Modo demo local

O atalho inicia o backend com:

```text
REPO52_DEMO_MODE=true
APP_ENV=development
REQUIRE_ALEMBIC_MIGRATIONS=false
```

Usa BD local ignorada: `web-dashboard/data.db` — exclua para redefinir linhas de demo.

## Banco de dados de produção

```text
DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:5432/quantum_command_52
DATABASE_POOL_MIN=1
DATABASE_POOL_MAX=10
JWT_SECRET=replace-with-at-least-32-random-characters
ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
RATE_LIMIT_AUTH_PER_MINUTE=12
RATE_LIMIT_API_PER_MINUTE=120
```

```powershell
cd web-dashboard
py -3.11 -m alembic upgrade head
```

## Use de verdade

1. Abra os arquivos CSV em `data`.
2. Adicione oportunidades, recursos, evidências ou itens reais.
3. Mantenha URLs oficiais nas colunas `source_url`.
4. Execute o script correspondente.
5. Use o Markdown/CSV gerado em `output`.
