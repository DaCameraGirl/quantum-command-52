## IBM Quantum

يتطلب السكربت الصارم V10 تبعيات مؤسسية ووصول IBM Runtime.

```powershell
py -3.11 strict_macro_quantum_v10.py --preflight
```

احتفظ بالأسرار في `.env`. لا تُلتزم مفاتيح IBM أو Alpaca أو قاعدة البيانات أو غيرها من المفاتيح الخاصة.

### بحث Alpaca الورقي

التداول المباشر محظور عمدًا. معاينة/إرسال ورقي فقط:

```powershell
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders --submit-paper-orders
```

## إعداد لوحة التحكم على الويب

```powershell
cd web-dashboard
npm install
py -3.11 -m pip install -r requirements.txt
py -3.11 server.py
```

طرفية ثانية:

```powershell
cd web-dashboard
npm run dev
```

افتح http://127.0.0.1:5173

## وضع العرض المحلي

يشغّل الاختصار الخلفية بـ:

```text
REPO52_DEMO_MODE=true
APP_ENV=development
REQUIRE_ALEMBIC_MIGRATIONS=false
```

يستخدم قاعدة بيانات محلية مُتجاهلة: `web-dashboard/data.db` — احذفها لإعادة تعيين صفوف العرض.

## قاعدة بيانات الإنتاج

```text
DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:5432/quantum_command_52
DATABASE_POOL_MIN=1
DATABASE_POOL_MAX=10
JWT_SECRET=replace-with-at-least-32-random-characters
ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
RATE_LIMIT_AUTH_PER_MINUTE=12
RATE_LIMIT_API_PER_MINUTE=120
```

```powershell
cd web-dashboard
py -3.11 -m alembic upgrade head
```

## استخدمه فعليًا

1. افتح ملفات CSV في `data`.
2. أضف فرصًا أو موارد أو أدلة أو عناصر حقيقية.
3. احتفظ بالروابط الرسمية في أعمدة `source_url`.
4. شغّل السكربت المطابق.
5. استخدم Markdown/CSV المُنشأ في `output`.
