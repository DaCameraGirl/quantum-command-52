## Schnellstart

Vom Repo-Stammverzeichnis in PowerShell:

```powershell
python grants.py rank
python housing_violations.py summarize
python shell_catalog.py estimate
python quantum_portfolio.py --capital 1000
python qml_signal_engine.py --capital 1000
python strict_macro_quantum_v10.py --preflight
```

Ausgaben landen im Ordner `output`.

## IBM Quantum

Das strikte V10-Skript erwartet Enterprise-Abhängigkeiten und IBM Runtime-Zugang.

```powershell
py -3.11 strict_macro_quantum_v10.py --preflight
```

Geheimnisse in `.env` aufbewahren. Keine IBM-, Alpaca-, Datenbank- oder anderen privaten Schlüssel committen.

### Alpaca-Paper-Recherche

Live-Trading ist absichtlich blockiert. Nur Paper-Vorschau/-Übermittlung:

```powershell
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders
py -3.11 strict_macro_quantum_v10.py --bankroll 1000 --preview-alpaca-orders --submit-paper-orders
```

## Lokaler Demo-Modus

Die Verknüpfung startet das Backend mit:

```text
REPO52_DEMO_MODE=true
APP_ENV=development
REQUIRE_ALEMBIC_MIGRATIONS=false
```

Verwendet ignorierte lokale DB: `web-dashboard/data.db` — löschen, um Demo-Zeilen zurückzusetzen.

## Produktionsdatenbank

```text
DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:5432/quantum_command_52
DATABASE_POOL_MIN=1
DATABASE_POOL_MAX=10
JWT_SECRET=replace-with-at-least-32-random-characters
ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
RATE_LIMIT_AUTH_PER_MINUTE=12
RATE_LIMIT_API_PER_MINUTE=120
```

```powershell
cd web-dashboard
py -3.11 -m alembic upgrade head
```

## Echten Einsatz

1. Öffnen Sie die CSV-Dateien in `data`.
2. Fügen Sie echte Chancen, Ressourcen, Nachweise oder Artikel hinzu.
3. Behalten Sie offizielle URLs in `source_url`-Spalten.
4. Führen Sie das passende Skript aus.
5. Nutzen Sie generiertes Markdown/CSV in `output`.
